В данной задаче предполагается, что на игровом поле 0 соответствует не заполненному квадрату, а размеры поля могут варьироваться (но поле остается квадратным).

Сделайте класс Game со следующей функциональностью:
1.Конструктор должен создавать объект игры нужного размера по вызову new Game(1,2,3,4,5,6,7,8,0) (количество аргументов различно для различных размеров игры).
2.Индексатор позволяет определить, какое значение находится в игре по координате [x,y]. 
3.Метод GetLocation(int value) позволяет определить, в какой ячейке находится переданное значение.
4.Метод Shift(int value) должен изменять состояние игры, передвигая фишку value на одно из соседних мест, где должен лежать 0. В случае, если 0 не находится на соседнем месте, должно возникать исключение. 

Требования:
1.Грамотная объектная декомпозиция и абстрагирование - разделение на классы, выделение полей и методов классов;
2.Обработка некорректных данных при создании объекта.
3.Использование конструкторов для создания объектов;
4.Корректное разбиение проекта на файлы (модули);
5.Соблюдение соглашений по именованию и форматированию кода: корректное именование модулей с кодом и заголовочных файлов, классов, методов и полей классов, переменных.
6.Индексатор и метод GetLocation должны работать за константное время.
7.Корректная обработка исключений.
